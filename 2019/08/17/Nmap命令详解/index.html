Nmap命令详解–Windows系统案例

====

Nmap是一款开源代码的工具，常用于网络探测和安全审核，基本的功能有三个：

一、探测主机是否在线

二、扫描主机端口

三、推断主机所用操作系统

因为我使用的系统是Windows的，所以使用的Zenmap，是Windows版本的，Nmap在Linux下的命令会有部分差异或是不能使用，所给命令总结是在Linux系统下进行的，例子是在Windows系统下的命令。

下面用172.16.13.39这个主机进行常用的Nmap命令演示。

Nmap入门

单个IP扫描

Nmap IP地址

例：Nmap 172.16.13.39

多个IP或网段扫描

Nmap IP IP …

例：Nmap 172.16.13.39 172.16.13.37

扫描相应网段IP

Nmap 开始IP-结束IP

例：Nmap 172.16.13.39-45

扫描IP所在网段IP

Nmap IP/子网掩码长度

例：Nmap 172.16.13.0/24

扫描主机列表targit.txt中所有的目标地址

Nmap -iL targets.txt //文件路径需使用绝对路径

例：Nmap -iL target.txt

扫描除了一个目标地址之外的所有地址

Nmap 网络号/子网掩码长度 -exclude 排除IP

例：Nmap –exclude 172.16.13.39 172.16.13.2-39

扫描除了某一个文件中的地址之外的所有地址

Nmap 网络号/子网掩码长度 –excludefile 文件名.txt //1.txt文件自己找个目录进行新建并保存任意数目的IP地址

例：Nmap –excludefile F:/1.txt 172.16.13.39 172.16.13.2-39

扫描指定IP的指定端口号

Nmap IP -p 端口号 //端口间用英文状态下的“，”分隔

例：Nmap -p 8080 172.16.13.39

路由追踪

Nmap –traceroute 目标IP

例：Nmap –traceroute 172.16.13.39

所在网段的在线状态

Nmap –sP IP地址/24

例：Nmap -sP 172.16.13.39/24 因软件原因会变成sn

目标地址的服务版本检测

Nmap –sV 目标IP

例：Nmap –sV 172.16.13.37

目标主机的操作系统

Nmap –O IP地址

例：Nmap –O 172.16.13.37

检测防火墙状态

Nmap –sF –T4 目标IP

例：Nmap –sF –T4 172.16.13.37

Nmap进阶

扫描主机或所在网段的弱口令

Nmap –script=auth IP

例：Nmap–script=auth 172.16.13.37

对主机的简单密码进行暴力破解

Nmap –script=bruth IP

因Windows系统版本的Zenmap不支持此命令，所以无截图。

扫描常见的漏洞

Nmap –script=vuln IP

例：Nmap –script vuln 172.16.13.39

更多服务开启情况

Nmap –n –p 端口号 –script=broadcast IP

例：Nmap -p 80 -n –script broadcast 172.16.13.39

因Windows系统版本的Zenmap不支持此命令，所以无截图。

whois解析

Nmap –script external 域名

例：Nmap –script external www.baidu.com
